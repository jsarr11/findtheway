Αλγόριθμος Kruskal

Ο αλγόριθμος Kruskal είναι ένας άπληστος αλγόριθμος για την εύρεση του Ελαχίστου Καλυπτικού Δέντρου (MST) ενός γράφου, επιλέγοντας τις ακμές με τη μικρότερη δυνατή τιμή και διασφαλίζοντας ότι δεν δημιουργούνται κύκλοι.

Βασική Ιδέα

  • Ταξινομούμε όλες τις ακμές του γράφου κατά αύξουσα σειρά βάρους.
  • Ξεκινώντας από την ελαφρύτερη ακμή, προσθέτουμε ακμές στο MST χωρίς να δημιουργούμε κύκλους.
  • Συνεχίζουμε μέχρι να συμπεριληφθούν Ν-1 ακμές, όπου Ν είναι ο αριθμός των κόμβων.

Περιγραφή Βημάτων

  1. Ταξινομούμε τις ακμές κατά αύξουσα σειρά βάρους.
  2. Ξεκινάμε με ένα κενό MST και ελέγχουμε κάθε ακμή:
    • Αν δεν δημιουργεί κύκλο, την προσθέτουμε στο MST.
    • Αν δημιουργεί κύκλο, την απορρίπτουμε.
  3. Επαναλαμβάνουμε το βήμα 2 μέχρι να έχουμε Ν-1 ακμές

Χαρακτηριστικά

  • Ο αλγόριθμος Kruskal επιλέγει πρώτα τις ελαφρύτερες ακμές, σε αντίθεση με τον Prim που ξεκινά από έναν κόμβο.
  • Είναι πιο αποδοτικός για αραιούς γράφους.
  • Χρησιμοποιεί συχνά τη δομή Disjoint Set (Union-Find) για την ανίχνευση κύκλων.

Kruskal algorithm

The Kruskal’s algorithm is a greedy algorithm used to find the Minimum Spanning Tree (MST) by always choosing the smallest available edge while ensuring no cycles are formed.

Basic Idea

  • Sort all edges in increasing order of weight.
  • Start with the smallest edge and keep adding edges to the MST as long as they do not form a cycle.
  • Stop when exactly N-1 edges are included, where N is the number of vertices.

Steps of the Algorithm

  1. Sort all edges in ascending order of weight.
  2. Start with an empty MST and process each edge:
    • If it does not form a cycle, add it to the MST.
    • If it forms a cycle, discard it.
  3. Repeat step 2 until the MST contains N-1 edges.

Characteristics

  • Kruskal’s algorithm picks the lightest edges first, unlike Prim’s, which starts from a single vertex.
  • It is more efficient for sparse graphs.
  • It often uses Disjoint Set (Union-Find) to efficiently detect cycles.
